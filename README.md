# Wiki-genshin
Projet: Création d'un site web de type wiki, pour les cours de bonnes pratiques.

Thématique: Inspiration de l'univers du jeu Genshin Impact.

L'arborecence est organisé pour avoir le backend, et le frontend dans deux dossier séparé, chaque fichier à une utilité, et suit un nomage conventionel.
Pour le nomage nous avons choisi d'utiliser la "camelCase convention", que ce soit pour nos fichier ou nos variable dans les fichiers.
Tout les codes présent sont complets, utiles, avec un fichier qui leurs est adapté. Les codes sont fait pour être le plus simple possible.

La base de données est hébergé en local avec MySQL, et le serveur tourne en localhost.
Les commit suivent la convention de git commit avec pour exemple: "[UPDATE] 'Modifying function b result in file a'"
La logique est continue au travers des differents dossier, chaque dossier à son utilité, et sa fonction.
Nous avons fait attention à ce que, le kiss dry, et le yagni soit respecté, pour être précis: pas de doublon au maximum de nos capacité, et pas de fonctionalité présente mais non utilisé.
